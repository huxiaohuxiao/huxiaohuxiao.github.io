Q: js里怎么去监听一个事件？监听方法的参数分别代表什么意思？其中哪些参数是关系到js事件流机制的？js事件流机制有哪些？工作原理是什么？

A: js的事件绑定有三种具体;事件监听则是addEventListener(event, function, useCapture)这个函数；其中第三个参数useCapture是关系到js的事件流机制；js的事件流机制有：事件捕获和事件冒泡；工作原理参考事件流机制

  • element.addEventListener(event, function, useCapture);
  • event: 必须, 字符串, 指定事件名
  • function: 必须。指定要事件触发时执行的函数
  • useCapture: 可选。布尔值，指定事件是否在捕获或冒泡阶段执行

请参考

  • js事件绑定
  • 事件流机制；

Q: 请详细写出AJAX的工作机制是怎么一个过程，列举JS处理跨域的常见方式并说明其原理.

A:

  1. Ajax的原理简单来说通过XmlHttpRequest对象来向服务器发异步请求，从服务器获得数据，然后用javascript来操作DOM而更新页面。
  2. JS的常见跨域处理方法
  • document.domain + iframe
  • 动态创建script
  • location.hash + iframe
  • window.name + iframe
  • postMessage（HTML5中的XMLHttpRequest Level 2中的API）
  • CORS
  • JSONP
  • web sockets

Q:在浏览器中输入一个网址之后会发生什么事情？请简述浏览器解析URL的过程，并根据这个原理给出前端上一些浏览器的优化解决方案。
A: 网上对于这个有很多种说法但是大体的思想都是差不多的，下面是其中我觉的还算ok的一种。优化在下面参考

  1. 域名的解析
  2. 浏览器查找该域名的 IP 地址
  3. 浏览器根据解析得到的IP地址向 web 服务器发送一个 HTTP 请求
  4. 服务器处理请求，并返回一个HTML页面服务器收到请求并进行处理
  5. 服务器返回一个响应
  6. 浏览器对该响应进行解码，渲染显示。
  7. 页面显示完成后，浏览器发送异步请求。
  8. 断开连接

Q: rem是什么？写出几种前端移动页面的适配方案，并说明前端在移动端开发和PC端开发有什么不同点，应该注意哪些事项？

A:

  1. rem: 是相对于根元素来设置字体大小的（IE6-8是不支持rem的）；
  2. 前端的适配方案
  • 固定高度，宽度自适应
  • 固定宽度，viewport缩放
  • rem做宽度，viewport缩放
  1. 前端在移动端开发和PC端开发有什么不同点
  • 布局：PC端咱们最常用的就是固定宽度980px（也有960，1000,1200），然后水平居中;移动端的话就需要响应式布局。
  • 内核（兼容性）：pc端的内核主流的就有五个Trident，Gecko/Firefox，WebKit，Presto，Blink，2013年4月，谷歌宣布采用Blink引擎替换Chrome的Webkit渲染引擎，同时Opera也宣布跟随谷歌采用Blink为渲染引擎；而移动端基本上只用考虑WebKit这个内核，因为目前移动端的主流浏览器的内核基本上都是基于WebKit或从WebKit修改过来的（国内的UC和QQ）。
  • 其他？（其他的有些地方也是有不同或者去别的，但是我觉得并没有这两个来的重要，如果有好的idea欢迎大家私我）
